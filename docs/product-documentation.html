1. Product Overview

The Nifty Options AI Agent is a fully automated options trading system for Indian markets (NSE). It scans NIFTY 50 index options in real-time, identifies high-probability trade setups using technical indicators and Open Interest analysis, sends trade alerts to the user's WhatsApp, waits for confirmation, executes orders on Zerodha via Kite Connect API, and manages the position with trailing stop-losses until exit.

What It Does (End to End)

1. Scan
Every 2 min, scan NIFTY option chain
Filter ATM ± 10 strikes
↓
2. Analyze
6 technical indicators detect trend
7 gates filter bad setups
↓
3. Score & Select
Rank options by OI, volume, trend
Pick the best setup (score ≥ 50)
↓
4. Alert
WhatsApp message with entry, SL, target
Wait for YES/NO (5 min timeout)
↓
5. Execute
Place market order on Zerodha
Set SL order automatically
↓
6. Monitor & Exit
Track every tick, trail SL
Auto-exit at target/SL/EOD

Key Capabilities

  • Real-time option chain scanning via KiteTicker WebSocket
  • Two trading strategies: Momentum Breakout + OI Reversal
  • 6-indicator trend detection engine
  • 7-gate entry filter system
  • 4-stage progressive trailing stop loss
  • Partial profit booking at 50% of target
  • Per-underlying parameter tuning (NIFTY vs BANKNIFTY)
  • WhatsApp alerts with confirmation workflow
  • Full backtesting engine with 1-min resolution
  • Interactive dashboard with trade drill-downs
  • AI-powered trade analysis via Claude API
  • Comprehensive risk management with hard limits

3. Live Trading System

3.1 Broker Integration (Kite Connect)

Authentication Flow (kite_auth.py)

1. Generate TOTP code using pyotp + KITE_TOTP_SECRET (base32) 2. POST credentials + TOTP to Zerodha login endpoint 3. Extract request_token from redirect URL 4. Exchange request_token for access_token via kite.generate_session() 5. Store access_token in SQLite (valid until ~6 AM next day) 6. On failure: retry 3 times with 30-second gaps, then WhatsApp alert

Trading Operations (kite_client.py)

Wrapper around KiteConnect providing: option chain fetching, LTP/quote retrieval, spot price, order placement (market/limit/bracket), order modification (for trailing SL), position tracking, and margin checking. All NFO instruments are cached once daily at startup.

WebSocket Streaming (kite_websocket.py)

KiteTicker in MODE_FULL provides: last_price, OHLC, volume, OI, bid/ask depth, and implied volatility. Runs in a background thread with auto-reconnect. Used for real-time position monitoring and tick-by-tick exit checking.

3.2 WhatsApp Integration

Outgoing Messages (twilio_client.py)

Sends formatted messages for: trade alerts, execution confirmations, exit notifications with P&L, daily summaries, error alerts, and system status (online/offline).

Incoming Commands (webhook_server.py)

Flask server receives Twilio POST webhooks. Supported commands:

4. Strategy Engine

4.1 Trend Detection — 6 Technical Indicators

The trend detector runs 6 independent indicators on 5-minute candles (even when using 1-min data for scanning, indicators are computed on resampled 5-min bars for stability). Each indicator votes BULL, BEAR, or NEUTRAL. The aggregate determines the trend.

1. EMA Crossover (5/13)

Exponential Moving Average of last 5 candles vs 13 candles. When fast EMA (5) crosses above slow EMA (13), it signals bullish momentum. Faster than traditional 9/21 to catch reversals earlier.

Signal: EMA5 > EMA13 = BULL EMA5 < EMA13 = BEAR

2. Price vs EMA 20

Is current price above or below its 20-period EMA? Shorter than EMA 50 (which lagged too much for detecting bearish trends).

Signal: Price > EMA20 = BULL Price < EMA20 = BEAR

3. RSI (14-period)

Relative Strength Index. Uses wider bands (55/45) so it contributes direction more often. Also used as an entry guard to block exhausted trends.

Signal: RSI > 55 = BULL RSI < 45 = BEAR 45-55 = NEUTRAL (0.5 bull vote)

4. MACD (12/26/9)

Moving Average Convergence Divergence histogram. Positive histogram means bullish momentum is accelerating.

Signal: Histogram > 0 = BULL Histogram < 0 = BEAR

5. Supertrend (7, 2.5)

Volatility-based trend indicator using ATR. Faster parameters (7, 2.5) instead of default (10, 3.0) for quicker reversal detection.

Signal: Price above line = BULL Price below line = BEAR

6. Price Action (3-candle slope)

Are the last 3 closing prices rising or falling? This is the only LEADING indicator (all others lag). Also used as the momentum confirmation gate.

Signal: Close[now] > Close[3 bars ago] = BULL Close[now] < Close[3 bars ago] = BEAR Otherwise = NEUTRAL

Aggregation Logic

bull_pct = (bullish_votes / 6) × 100 bear_pct = 100 - bull_pct NEUTRAL votes count as 0.5 bullish. Direction: bull_pct ≥ 60% → BULLISH (buy CE) bear_pct ≥ 55% → BEARISH (buy PE) otherwise → NEUTRAL (no trade) ASYMMETRIC THRESHOLDS: Bear needs only 55% (not 60%) because bearish signals are harder to generate from lagging indicators. By the time EMA/Supertrend/MACD all flip bearish, the RSI is already at 15-25 and the move is nearly over. EXHAUSTION FILTER: If bull_pct > 92% or bear_pct > 92%, SKIP. When ALL indicators agree, the trend is mature/ending, not beginning. Data showed losers had HIGHER trend strength (92%) than winners (69%).

4.2 The 7-Gate Filter System

Every potential trade must pass ALL 7 gates. Each gate was added to solve a specific problem discovered during backtesting.

GateCheckProblem It Solves
1 Trend DirectionBull ≥60% or Bear ≥55%Random entries in choppy markets (40% of original losses)
1b ExhaustionStrength < 92%Buying at end of moves. Losers had 92% strength vs winners 69%
2 RSI GuardCE: RSI < 75. PE: RSI > 209 losers entered with RSI > 65 in V1
3 MomentumLast 3 candles in trend direction33/42 losers hit SL in 5 min with MFE=0 — price was moving opposite
4 Daily LimitMax 2 trades/day per underlyingOvertrading — V0 did 6.7 trades/day, mostly noise
5 Cooldown30 min gap between tradesRevenge trading after a stop-loss
6 Risk CheckDaily P&L > -5K, positions < 2Catastrophic loss days

4.3 Strategy 1: Momentum Breakout

Logic: Buy options in the direction of the confirmed trend when a high-scoring, liquid option is found with volume > 10,000 contracts.

ENTRY: Trend must be BULLISH (buy CE) or BEARISH (buy PE) Option score ≥ 50/100 Volume > 10,000 contracts Premium between 50 and 500 STOP LOSS (ATR-based): SL = Entry - (ATR × multiplier × estimated_delta) Floor: never tighter than min_sl_pct of premium Cap: never wider than 50% of premium TARGET: Target = Entry + (risk × R:R ratio) NIFTY: R:R = 2.0 | BANKNIFTY: R:R = 1.5

4.4 Strategy 2: OI Reversal

Logic: When massive Open Interest (>200K contracts) builds up at a strike, option writers defend that level. High Put OI = support (bullish). High Call OI = resistance (bearish). Buy options where OI confirms the trend.

BULLISH trend + High Put OI at nearby strike: Writers sold PUTs = will defend support = buy CE BEARISH trend + High Call OI at nearby strike: Writers sold CALLs = will defend resistance = buy PE Same SL/target/sizing rules as Momentum. BACKTESTING RESULT: OI Reversal: 69-82% win rate, dominant P&L contributor Momentum: 20% win rate, net negative OI Reversal is the primary profit generator.

Position Sizing Formula

risk_per_unit = |Entry - SL| max_units = MAX_LOSS / risk_per_unit lots = floor(max_units / lot_size) lots = max(lots, 1) quantity = lots × lot_size Example: Entry=200, SL=170, lot=75 risk/unit = 30 max_units = 2000/30 = 66 lots = floor(66/75) = 0 → 1 qty = 75 max loss = 30 × 75 = Rs.2,250

6. Exit Logic

6.1 Exit Priority Order

#ConditionAction
1Price ≤ Trailing SLImmediate market exit (hard stop or trailing)
2EOD 3:15 PMSquare off all MIS positions
350% of target + qty > 1 lotPartial exit: sell 50% qty, trail SL to breakeven
4Price ≥ Target + held ≥ 10 candlesFull exit at target
5Trailing SL stages (below)Tighten SL progressively
6Held > 2 hours with < 10% profitExit (theta decay eating premium)

6.2 4-Stage Trailing Stop Loss

total_move = Target - Entry current_move = Current Price - Entry move_pct = (current_move / total_move) × 100 Stage 1: move_pct ≥ 30% (NIFTY) / 20% (BANKNIFTY) → Trail SL to BREAKEVEN (Entry price) → Trade is now risk-free Stage 2: move_pct ≥ 50% → Trail SL to Entry + 30% of current_move → Lock 30% of unrealized profit Stage 3: move_pct ≥ 70% → Trail SL to Entry + 50% of current_move Stage 4: move_pct ≥ 85% → Trail SL to Entry + 70% of current_move Example (Entry=200, SL=170, Target=260): total_move = 60 Price 218 (30%): SL 170 → 200 (breakeven) Price 230 (50%): SL → 200 + 30×0.3 = 209 Price 242 (70%): SL → 200 + 42×0.5 = 221 Price 251 (85%): SL → 200 + 51×0.7 = 235.7

8. Backtesting Engine

8.1 Data Sources

Synthetic Data

Generates realistic spot + option data without Kite API. Configurable: underlying, days, start price, candle interval (1-min or 5-min). Uses trending/mean-reverting regime cycles for realistic behavior. Automatically generates option chains around the price range with Black-Scholes-like pricing.

python -m backtest.run_backtest --synthetic --days 30 --interval 1

Real Kite Data

Fetches historical OHLCV from Kite Connect API. Auto-chunks requests (60-day limit for intraday). Caches to Parquet files. Builds option chain history from NFO instruments list.

python -m backtest.run_backtest --from 2025-11-01 --to 2025-11-30

8.2 Engine Architecture

The BacktestEngine replays historical candles through the exact same pipeline used in live trading: trend detection → 7-gate filter → strategy selection → position monitoring. When using 1-min data, indicators are computed on resampled 5-min bars for stability, while position monitoring uses the raw 1-min data for precise exit timing.

Key features: - Scan every N candles (default: 2 = every 2 minutes) - Monitor positions every candle (1-min resolution) - Indicators on 5-min resampled bars (no noise) - Full trade context captured (indicators, trend, scores) for dashboard - Per-underlying parameter overrides via MultiBacktestEngine - Parameter sweeps: --sweep sl_pct 0.10 0.50 0.05

10. AI Integration (Claude API)

Optional module (ai/trade_analyzer.py) that sends market snapshots to Claude Sonnet for analysis. Used when USE_AI_ANALYSIS=True in settings. Claude receives: spot price, trend data, top 10 options by score with Greeks, IV percentile, and risk parameters. It responds with structured JSON containing action, instrument, entry/SL/target, reasoning, and confidence score.

Also supports exit evaluation: given a position and current market data, Claude recommends HOLD, EXIT, or ADJUST with new SL/target.

System prompt enforces: JSON-only response, conservative bias (high-confidence trades only), mandatory SL/target, and clear reasoning (shown to user in WhatsApp alerts).
